# 盲変異 対照ペア + 鏡像汚染ガード（ゼロ段の着地手段）

## 背景（なぜこれが要るか）
レイテンシ議論から派生した長い設計往復の結論：
- **B（本文埋め込み→近傍で LLM 抽出をバイパス）は「凍結した A を照合で引くだけ」**。検知力は
  足さず、A の較正を凍らせ、新種の尾で盲になる。だから Bのゲートは「新種フォールバック率」の
  一段上に「**A は凍らせるに値するほど安定したか**」があり、その全部が単一の上流
  ボトルネック＝**ラベル取得（ゼロ段）** に律速される。
- ゼロ段は**道Bと正面で噛む**：山場（標的型/BEC）を "集める" と、パーソナライズ済みの生文面
  ——最も保存してはいけないデータ——を貯めることになる。だから "集める" でなく "生成する"。
- ただし攻撃生成と評価の**系統分離**を守らないと、身内で作って身内で採点する循環（H1-a の
  逆符号＝見せかけの高性能）になる。

## 採った設計：盲変異 対照ペア（(b) ＋ (a)の表層スライバのみ）
effective-benign（正規なのにレバーが立つ本物寄り）を土台に、target レバー **1個だけ** を
自然な一文で足した対を作る。対照ペアは極小 n で最大情報（文面を揃え1レバーだけ動かす＝
paired contrast で分散が潰れる）。ラベルは「動かしたレバー」という**構成事実**（外部コーパス
由来でも生成器の verdict でもない）。

素朴な「1レバー反転」は**鏡像構成**そのもので holdout 規律（鏡像汚染禁止）が禁じる。だから
妥当なのは**盲変異ペア**のみ：構成者は *どのレバーを動かすか* だけ決め、*表層の一文* は
逆算の匂いが出ないよう自然に書く（表層生成のみ攻撃側を借りてよい＝採点もラベル付けも
させない・系統分離）。その上で**二重ゲート**：
- (i) 盲でレバー再抽出し「target 指紋 *だけ* が動き他は不変」を機械確認 ← **本コミット**
- (ii) 設計に関与しない人間が挿入文を "自然" と読む ← コード外

## 実装（本コミット・純粋＝TDD 済み）
- `src/lib/contrastivePair.ts` — 純粋ガード。
  - `leverFingerprint` : score に効く全フィールドの正準指紋（intensity ドリフトも捕捉）。
  - `changedLevers` : 指紋差のレバー一覧＝"動いた"の定義。
  - `majorityVote` : K ドロー→レバーごと mode。**急所**：analyzeStructure は非決定なので
    単発でなく多数決。mode が僅差なら `fragile`（1回で裏返りうる＝要 K 増）。
  - `validateContrastivePair` : 「target だけ動き他不変」で valid。他ドリフト=contamination /
    target 不動=target_not_moved / 無変化=no_lever_moved / 有効ドロー不足=保留。
- `src/lib/__tests__/contrastivePair.test.ts` — 11 ケース緑（指紋・多数決・fragile・4種の
  invalid・保留）。LLM 非依存＝決定論。
- `scripts/validateContrastivePairs.ts` — LIVE ランナー（INTEGRATION ゲート）。base/mutated を
  analyzeStructure で K 回抽出→多数決→ガード。fragile は K 増、invalid は構成やり直しの合図。
- `src/data/contrastivePairs.ts` — データモジュール（**空**）＋型＋構成規律コメント。

## 残（ゼロ段の着地に要る人手）
effective-benign の土台を数通（実在名は架空化した "正規なのにレバーが立つ" 文）。それを土台に
盲変異ペアを構成→`validateContrastivePairs.ts` でガード通過→採用、で**ゼロ段が着地**する。

## 提出（7/10）の誠実な線
- **前座**＝実データで即答（量産フィッシング・本物）
- **山場**＝合成対照ペアで**機構**を示す（標的型二層：稀な一通は敢えて遅く推論に逃がす）
- **本物の尾**＝今後

デモ主役は「速さ先行＋"敢えて遅い"を山場に」の一本の弧。48秒はここで弱点から仕様に変わる。
盲変異ペアは、この "機構を示す" を**循環で偽らず**成立させる唯一の作り方。
