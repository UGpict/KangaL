# Google OAuth 審査 — 実務手順（gmail.readonly / 制限付きスコープ）

サービス: KangaL ／ project `ai-bridging`(番号 `649847191589`) ／ 本番 URL `https://kangal-649847191589.us-central1.run.app`

前提の事実: `gmail.readonly` は **制限付きスコープ（restricted）**。一般公開の完全承認には OAuth 同意画面の公開審査＋**第三者セキュリティ審査（CASA）**が要り、**数週間〜数か月**かかる。1週間で「承認」は出ない。だから運用は下の2本立て。

---

## A. 今すぐ効く：テストユーザーで実ユーザーに使わせる（審査不要）

同意画面が **テストモード**なら、登録した最大100人が審査なしで `gmail.readonly` をフルに使える（「未確認アプリ」警告を1回クリックするだけ）。

手順（Google Cloud Console・人間が実行）:
1. Console → **APIs & Services → OAuth consent screen**（対象 project = `ai-bridging`）。
2. Publishing status が **Testing** であることを確認（Production になっていたら Testing に戻すか、テストユーザー欄の有無を確認）。
3. **Test users → + ADD USERS** に、試してほしい人／審査員のメールアドレスを追加（最大100）。**Save**。
4. 追加した各人は `https://kangal-649847191589.us-central1.run.app/` で Gmail 連携 → 「未確認アプリ」画面で「詳細 → 安全でないページに移動」→ consent → 自分の受信箱で判定を体験できる。

これで「（登録した）誰でも実 Gmail で動く」が今週中に成立する。審査員には「実 Gmail で動作＋公開審査は申請中」と正直に見せられる。

- [ ] **OAuth 同意画面**に：アプリ名 / ユーザーサポートメール / アプリロゴ / アプリのホームページ / プライバシーポリシー URL / 利用規約 URL / 承認済みドメイン を設定。
- [ ] **スコープの正当化文**（下記）＋ **デモ動画**（スコープの使い道を映す）を用意して申請。
- [ ] **CASA セキュリティ審査**（restricted の本丸・時間と費用）。ここは提出後にずれる前提。

### スコープ正当化文（申請フォーム用・そのまま貼れる）

> KangaL は受信メールの詐欺リスクを判定し、その理由をユーザーに説明する詐欺検知アプリです。判定には、メールの件名・本文・送信者・認証ヘッダ（SPF/DKIM/DMARC）を読み取る必要があるため、`gmail.readonly` を要求します。書き込み系スコープは要求しません（送信・変更・削除は行いません）。読み取ったメール内容はその場での詐欺判定にのみ使用し、サーバーには保存しません。アクセストークンはユーザーのブラウザ Cookie に暗号化して保持し、サーバーには保存しません。Google API から取得したデータの使用は Google API Services User Data Policy（Limited Use 要件を含む）に準拠します。

### データ扱いの実装事実（審査で有利・そのまま説明に使える）

- アクセストークンは **AES-256-GCM で暗号化し httpOnly/Secure Cookie にのみ**保存。サーバー・DB・ログに保存しない（`src/lib/gmailSession.ts`）。
- メール本文は**保存せず**、判定時にその場で解析して破棄。学習用コーパスにも実メールを書かない（読み取り専用境界）。
- 第三者送信は判定処理のための Vertex AI(Gemini)/Web Risk/RDAP のみ。販売・広告目的なし。
