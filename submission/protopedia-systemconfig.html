Gmail と連携し、受信メールを自動で取得・判定します（手で貼り付ける必要はありません）。判定は本番 Cloud Run 上の防御パイプラインが担います。

防御パイプライン（本番・Cloud Run）

  1. 構造分解：Gemini（Vertex / 2.5 Flash）がメッセージから6レバー（緊急性・権威・報酬/恐怖・誘導・個人化・孤立化）を逆算。
  2. 能動調査（function-calling）：matchKnownScams（事例DB照合）を必ず1回。加えて Web Risk（URL評価）／RDAP（ドメイン年齢）／SPF・DKIM・DMARC（送信者認証）／公的アラート照合を、入力の特徴に応じてモデルが動的に選択。
  3. 総合判断：レバー×重みの素点に調査の危険シグナルを加点のみで重ね、確信度スコアと「なぜ危ないか」の日本語説明を生成。
  4. 自己改善：すり抜けた型を Firestore に書き戻し、次の照合材料に。

攻防ループ（自己改善）

攻撃AI（狼）が検知器の死角を突く「6レバーの型」を進化 → 防御AI（番犬）が同じパイプラインで判定 → すり抜けた型を事例DBへ書き戻し → 次ラウンドで matchKnownScams が類似を捕捉。ラウンドを回して検知器が自分で強くなります。

安全設計

そのまま送れる完成詐欺文は生成も保存もしません。詐欺は「6つの心理レバーの設定値」として表現するので、悪用可能な危険物が残りません。

インフラ

Next.js（TypeScript）／ Cloud Run ／ Firestore ／ Vertex AI（Gemini 2.5 Flash）／ Google Web Risk ／ RDAP。鍵レス（ADC）でデプロイし、鍵は Secret Manager 経由で管理。サービスアカウント JSON はリポジトリにもイメージにも置きません。

公開デモの多層防御

公開の判定APIは IP 単位＋インスタンス単位の2層レート制限で保護（Cloud Run の静的天井＝max-instances／concurrency／timeout・予算アラートと多層）。Gemini の構造化出力は responseSchema＋実行時の深い検証（enum・数値範囲・未知キー拒否）の二重壁で受け、不正応答は判定保留に倒します。説明生成の AI が停止しても、検出レバーから決定論的に「なぜ危ないか」を説明します。
